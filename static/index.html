4. OpenFoodTox query

Question EFSA OpenFoodTox records from either a local SQLite index built from the official Excel export or, for institutions, a local IUCLID 6 import. Use this step to look for toxicant identity records, toxicological values/study documents, and publication links that can contextualize the selected AOP evidence.

OpenFoodTox is not queried from a public EFSA API. Regular clients can run scripts/import_openfoodtox.py --download-latest --db data/openfoodtox.db to download and index the official Excel export automatically. Institutions can alternatively import i6z dossiers into IUCLID 6 and configure the local IUCLID REST API URL.

5. Quantitative screening

Compare a NAM point of departure with an exposure estimate. This is a prototype screening calculator; it does not provide validated regulatory thresholds.

Quantitative outputs are only as reliable as the submitted POD, exposure, and IVIVE inputs. Use for prioritization and hypothesis refinement unless validated sources and program-specific thresholds are configured.
